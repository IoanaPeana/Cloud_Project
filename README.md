## Architecture Overview

```
Internet
   │
   ▼
┌─────────┐        ┌──────────────┐      ┌───────────────┐
│  Nginx  │──────▶│     App      │─────▶│  HashiCorp    │
│  :80    │        │  (Flask:5000)│      │  Vault :8200  │
│ (Proxy) │        │  internal    │      │  internal     │
└─────────┘        └──────────────┘      └───────────────┘
                                                   ▲
                        ┌──────────────────────────┘
                        │ secrets injected at runtime

┌─────────────┐    ┌──────────────┐    ┌──────────────┐
│node-exporter│───▶│  Prometheus  │───▶│   Grafana    │
│  internal   │    │  :9090       │    │  :3000       │
└─────────────┘    └──────────────┘    └──────────────┘
```

**All services share an internal Docker bridge network. Only ports 80 and 3000 are exposed.**

## What `setup.sh` Does (Automatically)

| Step | Action |
|------|--------|
| 1 | Installs Docker if not present |
| 2 | Generates a random Vault root token (via `openssl rand`) |
| 3 | Writes token to `.env` (not committed to git) |
| 4 | Builds and starts all 6 containers via `docker compose up -d` |
| 5 | Waits for Vault to be ready (health-check loop) |
| 6 | Enables KV-v2 secrets engine in Vault |
| 7 | Generates a random API key and stores it in Vault |
| 8 | Restarts the app so it fetches the new secret |
| 9 | Verifies the app responds through Nginx |

## Security Compliance

- ✅ **No hardcoded secrets** — Vault token generated by `openssl rand` at runtime
- ✅ **Network isolation** — App container has no public port; only Nginx is exposed
- ✅ **Secrets via API** — App calls Vault HTTP API (`/v1/secret/data/myapp`) at request time
- ✅ **`.env` excluded from git** — listed in `.gitignore`

## Component Details

### HashiCorp Vault
- Runs in dev mode with a randomly generated root token
- KV-v2 secrets engine enabled at path `secret/`
- Stores `api_key` under `secret/myapp`

### Web Application (Flask)
- Reads `VAULT_ADDR` and `VAULT_TOKEN` from environment variables only
- Calls Vault HTTP API on every request to retrieve the current `api_key`
- No passwords or secrets in source code

### Nginx
- Sole entry point on port 80
- Proxies all traffic to the app on the internal network
- Security headers included (`X-Frame-Options`, `X-Content-Type-Options`)

### Prometheus + Node Exporter
- Scrapes VM CPU, RAM, and system load every 15 seconds

### Grafana
- Auto-provisioned datasource (Prometheus) and dashboard (VM Metrics)
- Shows CPU %, RAM used/total, system load — all live and auto-refreshing
